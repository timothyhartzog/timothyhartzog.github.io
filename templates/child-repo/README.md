# Subdomain App Template — Hartzog.ai

This template sets up a new subdomain application that integrates with the Hartzog.ai ecosystem.
Each app deploys to Azure Static Web Apps as `<app-name>.hartzog.dev` with GitHub Pages as a backup.

## Quick Start

1. **Copy these files** into your new repository
2. **Edit `.hartzog.json`** — set your app name, description, and subdomain
3. **Choose your framework** — React 19 (default), or any framework that builds to static files
4. **Set up Azure SWA** — see Infrastructure section below
5. **Register your app** — add an entry to `src/theme/sites.json` in the main `timothyhartzog.github.io` repo

## Shared Theme

The theme is loaded via CDN (jsDelivr serves directly from the main GitHub repo):

```html
<!-- CSS variables and base styles -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/timothyhartzog/timothyhartzog.github.io@main/src/theme/base.css" />

<!-- Navigation bar (Web Component) -->
<script src="https://cdn.jsdelivr.net/gh/timothyhartzog/timothyhartzog.github.io@main/src/theme/nav-bar.js" defer></script>

<!-- Footer (Web Component) -->
<script src="https://cdn.jsdelivr.net/gh/timothyhartzog/timothyhartzog.github.io@main/src/theme/footer.js" defer></script>
```

All `--hz-*` CSS variables are available for consistent styling. See `src/theme/colors.css` for the full palette.

## Infrastructure

### Azure Static Web Apps (Primary)

```bash
# Create the SWA resource
az staticwebapp create \
  --name hartzog-ai-<app-name> \
  --resource-group hartzog-ai-rg \
  --source https://github.com/timothyhartzog/<repo-name> \
  --branch main \
  --app-location "/" \
  --output-location "dist" \
  --sku Free

# Add custom subdomain
az staticwebapp hostname set \
  --name hartzog-ai-<app-name> \
  --resource-group hartzog-ai-rg \
  --hostname <app-name>.hartzog.dev
```

Then add a CNAME record: `<app-name>.hartzog.dev → <auto-generated>.azurestaticapps.net`

### GitHub Pages (Backup)

Enable GitHub Pages in repo settings pointing to the `gh-pages` branch or GitHub Actions.

### GitHub Actions Workflow

Add `.github/workflows/deploy.yml`:

```yaml
name: Deploy

on:
  push:
    branches: [main]
  pull_request:
    types: [opened, synchronize, reopened, closed]
    branches: [main]

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: npm
      - run: npm ci
      - run: npm run build
      - uses: actions/upload-artifact@v4
        with:
          name: site-build
          path: dist

  deploy-azure:
    needs: build
    if: github.event_name == 'push' || (github.event_name == 'pull_request' && github.event.action != 'closed')
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/download-artifact@v4
        with:
          name: site-build
          path: dist
      - uses: Azure/static-web-apps-deploy@v1
        with:
          azure_static_web_apps_api_token: ${{ secrets.AZURE_STATIC_WEB_APPS_API_TOKEN }}
          repo_token: ${{ secrets.GITHUB_TOKEN }}
          action: upload
          app_location: /
          output_location: dist
          skip_app_build: true

  close-preview:
    if: github.event_name == 'pull_request' && github.event.action == 'closed'
    runs-on: ubuntu-latest
    steps:
      - uses: Azure/static-web-apps-deploy@v1
        with:
          azure_static_web_apps_api_token: ${{ secrets.AZURE_STATIC_WEB_APPS_API_TOKEN }}
          action: close

  deploy-github-pages:
    needs: build
    if: github.event_name == 'push'
    runs-on: ubuntu-latest
    environment:
      name: github-pages
    steps:
      - uses: actions/download-artifact@v4
        with:
          name: site-build
          path: dist
      - uses: actions/upload-pages-artifact@v3
        with:
          path: dist
      - uses: actions/deploy-pages@v4
```

## Registering Your App

Add an entry to `src/theme/sites.json` in the main repo:

```json
{
  "name": "Your App Name",
  "url": "https://app-name.hartzog.dev",
  "repo": "your-repo-name",
  "description": "What this app does",
  "icon": "code",
  "primary": false,
  "type": "app",
  "subdomain": "app-name",
  "hosting": "azure-swa"
}
```

## .hartzog.json Flag File

The `.hartzog.json` file signals that this repo is part of the Hartzog.ai ecosystem:

```json
{
  "parent": "https://www.hartzog.dev",
  "theme": true,
  "name": "App Name",
  "description": "Short description",
  "subdomain": "app-name",
  "type": "app",
  "hosting": "azure-swa"
}
```

The `<hartzog-nav>` Web Component reads the `parent` attribute to show a back link to the main site.

| Web Components | Shared nav, footer, forms |
