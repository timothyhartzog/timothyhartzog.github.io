# Azure Static Web Apps Infrastructure

## Architecture

```
www.hartzog.ai          → Azure SWA (main site, this repo)
app1.hartzog.ai         → Azure SWA (subdomain app, separate repo)
app2.hartzog.ai         → Azure SWA (subdomain app, separate repo)
timothyhartzog.github.io → GitHub Pages (backup mirror of main site)
```

## Setup

### 1. Create Azure Static Web App

```bash
az staticwebapp create \
  --name hartzog-ai-main \
  --resource-group hartzog-ai-rg \
  --source https://github.com/timothyhartzog/timothyhartzog.github.io \
  --branch main \
  --app-location "/" \
  --output-location "dist" \
  --sku Free
```

### 2. Configure Custom Domain

```bash
# Add custom domain
az staticwebapp hostname set \
  --name hartzog-ai-main \
  --resource-group hartzog-ai-rg \
  --hostname www.hartzog.ai

# DNS records to add:
# www.hartzog.ai  → CNAME → <auto-generated>.azurestaticapps.net
# hartzog.ai      → A     → Azure SWA IP (or redirect to www)
```

### 3. Get Deployment Token

```bash
az staticwebapp secrets list \
  --name hartzog-ai-main \
  --resource-group hartzog-ai-rg
```

Add the `apiKey` value as `AZURE_STATIC_WEB_APPS_API_TOKEN` in GitHub repo secrets.

### 4. Create Subdomain App

For each subdomain application:

```bash
az staticwebapp create \
  --name hartzog-ai-<app-name> \
  --resource-group hartzog-ai-rg \
  --source https://github.com/timothyhartzog/<app-repo> \
  --branch main \
  --app-location "/" \
  --output-location "dist" \
  --sku Free

az staticwebapp hostname set \
  --name hartzog-ai-<app-name> \
  --resource-group hartzog-ai-rg \
  --hostname <app-name>.hartzog.ai
```

## DNS Configuration

| Record | Type  | Value                                    |
|--------|-------|------------------------------------------|
| www    | CNAME | `<swa-hostname>.azurestaticapps.net`     |
| @      | A     | Azure SWA IP (or URL redirect to www)    |
| app1   | CNAME | `<app1-swa-hostname>.azurestaticapps.net`|
| app2   | CNAME | `<app2-swa-hostname>.azurestaticapps.net`|

## Features

- **Free SSL** for all custom domains (auto-provisioned)
- **Global CDN** included with Azure Front Door
- **PR preview environments** created automatically
- **GitHub integration** for CI/CD out of the box
- **Fallback routing** configured in `staticwebapp.config.json`
